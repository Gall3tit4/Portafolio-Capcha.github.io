React

Resumen de la Semana


React es una biblioteca de JavaScript creada por Facebook en 2011, diseñada para construir interfaces de usuario reutilizables, especialmente en aplicaciones de una sola página (SPA). Uno de sus conceptos clave es el DOM virtual, el cual permite realizar cambios en memoria antes de aplicarlos al DOM real del navegador, mejorando así el rendimiento.

React

Fundamentos de React

React surgió en Facebook como solución para construir interfaces dinámicas y eficientes, especialmente para Single Page Applications (SPA). Su arquitectura basada en componentes permite crear elementos UI reutilizables que gestionan su propio estado, revolucionando el desarrollo frontend moderno.

Mecanismos Internos

El corazón de React es su DOM virtual, una representación ligera del DOM real que permite calcular diferencias eficientemente (diffing algorithm) y aplicar solo los cambios necesarios al navegador. Este enfoque minimiza manipulaciones costosas del DOM, mejorando significativamente el rendimiento.

Herramientas del Ecosistema

El ecosistema React ofrece herramientas como Create React App (para configuraciones iniciales), Vite (para desarrollo ultrarrápido) y Next.js (para aplicaciones SSR/SSG). Estas abstraen configuraciones complejas de Webpack y Babel, permitiendo enfocarse en la lógica de la aplicación.

Arquitectura de Proyectos

Un proyecto típico organiza el código en carpetas como src/ (para componentes) y public/ (para assets estáticos). Los archivos clave como main.jsx (punto de entrada) y App.jsx (componente raíz) establecen la estructura base que Webpack procesa para producción.

Implementación Flexible

Para proyectos simples, React puede integrarse via CDN sin necesidad de herramientas de compilación, aunque esta aproximación pierde ventajas como el Hot Module Replacement. La documentación oficial detalla estos escenarios y ofrece guías para migrar a configuraciones más robustas.

Consideraciones de Desarrollo

El servidor de desarrollo incluido (vite dev server o webpack-dev-server) proporciona funcionalidades como hot reloading, pero no está diseñado para entornos productivos. Para deployment, se requiere construir assets optimizados que servidores como Nginx puedan entregar estáticamente.

Funciones Clave de React

useState()

Gestión de estado local en componentes funcionales:

const [count, setCount] = useState(0);

Permite almacenar y actualizar datos que persisten entre renderizados.

useEffect()

Manejo de efectos secundarios:

useEffect(() => {
                                    // Lógica de efecto
                                    return () => cleanup();
                                    }, [dependencies]);

Reemplaza ciclos de vida de componentes clase con mayor control.

useContext()

Acceso al contexto sin prop drilling:

const user = useContext(UserContext);

Facilita compartir datos globales en el árbol de componentes.

useReducer()

Gestión de estado complejo:

const [state, dispatch] = useReducer(reducer, initialState);

Patrón similar a Redux para flujos de estado predecibles.

Renderizado Condicional

Técnicas fundamentales para UI dinámica:

  • Operadores lógicos: {isLoggedIn && }
  • Ternarios: {isLoading ? : }
  • Early return: Bloqueo de renderizado con condiciones

¿Qué Aprendí?

Aprendí los fundamentos esenciales de React, una biblioteca de JavaScript diseñada para construir interfaces de usuario dinámicas y eficientes. Comprendí cómo funciona el DOM virtual para actualizar solo los elementos necesarios en pantalla, lo que mejora el rendimiento de las aplicaciones. Aprendí a crear componentes reutilizables que encapsulan lógica y estructura visual, y entendí cómo se estructuran los proyectos en React utilizando archivos como App.jsx y main.jsx. También me familiaricé con herramientas modernas como Create React App, Vite y Next.js, que simplifican la creación y gestión de proyectos.

¿Como Aprendí?

Aprendí mediante explicaciones teóricas en clase, apoyadas con ejemplos prácticos y actividades guiadas que me permitieron construir pequeños componentes en React. Exploré proyectos reales, modifiqué estructuras, y observé cómo los cambios en el código se reflejaban dinámicamente en la interfaz. También reforcé mi aprendizaje revisando la documentación oficial de React, utilizando editores como Visual Studio Code y probando herramientas como Create React App para generar entornos listos para trabajar. La práctica constante me ayudó a entender la lógica de los componentes, la gestión de estados y el flujo de datos en React.
